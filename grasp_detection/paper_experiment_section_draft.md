# Section 4: Experimental Evaluation

In this section, we present a quantitative and qualitative evaluation of the proposed framework to verify its effectiveness in improving target-region grasping precision for transparent bagged items. 

## 4.1 Experimental Setup
The evaluation was conducted on a Linux server equipped with an Intel Xeon processor and an NVIDIA GeForce RTX 4090 GPU (24 GB VRAM) for deep neural network inference. We employ two representative state-of-the-art (SOTA) 6-DoF grasp detection models as backbones:
1. **AnyGrasp**: A PointNet-based commercial grasping framework utilizing the GraspNet-1Billion benchmark backend.
2. **Contact-GraspNet**: A PointNet++ based open-source grasping model developed by NVIDIA. We run an open-source PyTorch implementation of Contact-GraspNet to bypass compatibility issues under CUDA 12.4.

Both backbones utilize their official pre-trained weights without any fine-tuning or domain adaptation, demonstrating the plug-and-play capability of our proposed preprocessing module.

## 4.2 Dataset and Evaluation Protocol
The test dataset consists of 21 independent physical scenes containing transparent bagged products collected via a real-world camera system. Each sample includes a high-resolution RGB image, a 16-bit depth map (measured in millimeters), camera intrinsics ($K$), and a high-precision binary target segment mask independently generated by the Segment Anything Model 2 (SAM 2.1 Large).

To evaluate the precision of the generated grasps, we define the **Target Mask Precision** metric. For a set of generated 6-DoF grasp candidates, we project their 3D coordinates back into the 2D image plane using the camera intrinsics:
$$u = \text{round}\left(\frac{x \cdot f_x}{z} + c_x\right), \quad v = \text{round}\left(\frac{y \cdot f_y}{z} + c_y\right)$$
where $(x, y, z)$ represents the 3D position in the camera coordinate frame, and $(u, v)$ represents the projected pixel coordinates. The Target Mask Precision is computed as the fraction of projected coordinates that fall within the positive region of the SAM 2.1 target mask. 

It is crucial to clarify that the Target Mask Precision measures **geometric localization precision** (i.e., whether the grasp proposals target the physical boundary of the object of interest rather than the background table or walls) and does not directly equate to the physical robot grasp success rate. However, a high localization precision is a fundamental prerequisite for avoiding collisions and ensuring a stable physical grasp.

## 4.3 Baselines and Ablation Groups
We evaluate the performance across the following configuration groups:
- **AnyGrasp Group B (Original Depth)**: The baseline AnyGrasp model directly processing the raw depth map and RGB image without any target-region preprocessing.
- **AnyGrasp Group D (Strict Qd+SAM2)**: AnyGrasp model equipped with our proposed depth quality map ($Q_d$) filtering and SAM2 mask constraint. This group uses a hard-crop constraint where all depth values outside the SAM2 mask are set to zero, forcing the model to generate grasps purely within the target region.
- **CGN-Native**: The baseline Contact-GraspNet model operating on the full scene point cloud without any mask constraints or workspace cropping.
- **CGN-Mask**: The Contact-GraspNet model utilizing the target mask to extract a 3D local region (`local_regions=True`) and filtering the generated grasp contacts to lie strictly on the segmented surface (`filter_grasps=True`).

## 4.5 Cross-Architecture Generalization Analysis
The results demonstrate a robust generalization capability of our proposed preprocessing module across fundamentally different grasp detector architectures. For AnyGrasp, which operates directly on the cropped depth map, the Target Mask Precision reaches a perfect $1.0000$. More importantly, for Contact-GraspNet, which uses contact-based representation and is prone to background noise, the introduction of the mask constraints improves the precision of the physical contact points from $0.1905$ to $0.8571$ (a **4.5-fold increase**). This confirms that our framework is detector-agnostic and acts as a generic front-end enhancement layer for 3D grasp models.

## 4.6 Palm Center vs. Contact Point Analysis
We observe a significant discrepancy in the absolute precision values of Contact-GraspNet depending on the evaluation reference point. Specifically, under the CGN-Mask configuration, the precision computed using the **Palm Center** is only $0.1747$, while the precision computed using the **Contact Point** reaches $0.8571$. 

This discrepancy is explained by the physical definition of the grasp coordinate frames. In Contact-GraspNet, the 6-DoF grasp translation vector ($T_{translation}$) represents the geometric center of the gripper's hand palm. To prevent the physical gripper fingers from colliding with the target during approach, the palm center is offset backward from the physical contact point on the object's surface by approximately $5 \sim 8\text{cm}$ along the approach axis. 

In our camera setup, which views the scene from an oblique top-down perspective, this 3D backward offset translates to an upward pixel shift in the 2D projection plane. Consequently, the projected palm center pixel lands outside the 2D boundary of the SAM2 mask (landing on the background table or upper regions), artificially penalizing the precision score. By utilizing the predicted **Contact Point** ($P_{contact}$) as the projection reference, we eliminate the visual offset introduced by the gripper's physical thickness, accurately reflecting the model's actual ability to locate and contact the object surface.

## 4.7 Failure Case Analysis
Qualitative selection shows that in certain extreme circumstances, such as in `sample_12` and `sample_13`, the proposed mask constraints did not result in a successful grasp generation. These failure modes are characterized by:
- **Severe depth voiding**: In these scenes, the transparent bag material combined with strong external light reflections caused the depth camera to fail completely in obtaining depth measurements inside the bag region, resulting in a large cluster of zero-depth (void) pixels.
- **Empty local point clouds**: When the local box was cropped using the SAM2 mask, the resulting local point cloud contained zero or too few valid points. Consequently, Contact-GraspNet failed to generate any grasp proposals, yielding a precision of $0.0000$ and a grasp candidate count of $0$.

These results indicate that the framework remains dependent on the baseline point cloud density and does not generate virtual geometries to fill massive depth voids.

## 4.8 Limitations
While the proposed framework achieves statistically significant improvements in target localization precision, several limitations remain to be addressed:
1. **2D Projection Evaluation**: The Target Mask Precision is evaluated using 2D image projections. Although we corrected the gripper thickness bias using contact point evaluation, it does not fully capture the 3D alignment and physical contact dynamics between the gripper and the irregular target.
2. **Candidate Count Shrinkage**: Restricting the grasp search space to the target mask region significantly reduces the number of candidate grasp proposals (e.g., from 50 to 15.5 for Contact-GraspNet). In cluttered scenes with highly constrained gripper approach directions, this reduction in candidate count may limit the motion planner's ability to find a collision-free path.
3. **Absence of Physical Disturbance**: The evaluation does not model the physical material deformation of the plastic bags during gripper contact, nor does it account for mechanical friction. Real-world robotic validation remains necessary to quantify how the improved geometric localization precision translates to actual physical grasp success rates.
